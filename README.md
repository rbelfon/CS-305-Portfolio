## Reflection

Artemis Financial is a financial consulting company that develops personalized financial plans for its clients, including savings, retirement, investments, and insurance. The company wanted to modernize its software while improving its security. My responsibility was to identify security vulnerabilities, recommend solutions, and refactor the application to use secure coding practices that would better protect customer data.

One thing I did well during this project was identifying security vulnerabilities and understanding how they could affect the application. I also learned that software security is much more than simply fixing bugs. Secure coding helps protect sensitive information, reduces the risk of cyberattacks, and increases customer confidence. Strong security practices also help a company avoid financial loss, downtime, and damage to its reputation.

The most challenging part of the vulnerability assessment was working with the OWASP Dependency-Check tool and troubleshooting issues with the National Vulnerability Database (NVD). Although the external service prevented the scan from completing successfully, the experience helped me better understand how vulnerability scanning tools work and why keeping software dependencies current is an important part of secure software development.

I increased the application's security by implementing HTTPS with an SSL certificate to encrypt communications and by adding a SHA-256 checksum to verify data integrity. These changes added multiple layers of protection to the application. In future projects, I would continue using tools such as OWASP Dependency-Check, secure code reviews, vulnerability assessments, and industry security guidelines to identify risks and determine the best mitigation strategies.

After refactoring the application, I verified that it remained functional by compiling and running the project successfully and testing the application over HTTPS. I also attempted to perform a secondary static security scan using the OWASP Dependency-Check Maven plugin to check whether any new vulnerabilities had been introduced during the refactoring process.

Throughout this course I used several resources and tools that will be valuable in future projects, including Eclipse, Maven, Spring Boot, Java Keytool, the OWASP Dependency-Check plugin, GitHub, and industry security best practices. I also gained experience using SHA-256 hashing, SSL certificates, HTTPS configuration, and secure coding techniques to strengthen software applications.

If a future employer asked for an example of my work, I would share my completed Artemis Financial Practices for Secure Software Report. It demonstrates my ability to analyze software security requirements, implement secure communications, refactor code using secure coding practices, test applications, and document my work in a professional manner.
